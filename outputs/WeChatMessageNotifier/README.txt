微信消息提醒器
==============

版本：2.0.0

初始代码与文档由 OpenAI Codex 于 2026-06-29 根据项目所有者需求编写。

功能
----
- 完全本地运行，不联网，不消耗 Token。
- 只读监控微信 4.x 当前会话列表。
- 新消息出现时在屏幕右上角显示联系人和消息摘要。
- 弹窗显示 2 分钟后自动关闭；点击弹窗可打开微信并跳转到对应会话。
- 弹窗右上角的“×”只关闭当前提醒，不会打开微信。
- 消息会同时静默保存到 Windows 通知中心，不额外弹出系统横幅。
- 如果自定义弹窗失败，Windows 系统横幅会自动接管提醒。
- 自定义弹窗关闭后通知中心记录仍然保留，点击记录可打开微信。
- 清空通知中心后，下一条消息从新的通知序列开始。
- 同一联系人连续发来消息时更新现有弹窗，避免重复堆叠。
- 右下角的 QQ、系统短通知或托盘展开面板遮挡时，微信提醒会自动上浮；剪贴板历史等大型面板不会触发上浮。
- 上浮和回落使用约 60 FPS 的非线性阻尼弹簧动画。
- Windows 11 下使用圆角、轻透明和瞬态亚克力背景。
- 点击提醒打开微信时保留原窗口状态；最大化的微信不会被降为普通窗口。
- 微信处于前台时仍会弹窗，避免点击上一条消息后漏掉其他会话的新消息。
- 使用微信会话 AutomationId 和未读数量识别新消息，忽略时间文本及短暂不完整扫描造成的重复提醒。
- 同一联系人和摘要在 5 分钟内只提醒一次，防止未读计数回跳造成二次提醒。
- 不读取微信数据库，不注入微信进程，不记录联系人或消息正文。
- 单个日志达到 1 MB 时自动轮转，最多保留 3 份历史日志。

使用
----
1. 保持微信已登录。
2. 双击 WeChatMessageNotifier.exe。
3. 第一次启动只建立当前会话基线，不会把旧消息全部弹出。
4. 右键系统托盘中的提醒器图标，可暂停、启用隐私模式、发送测试通知或退出。
5. 点击消息弹窗会打开微信并跳转到对应的一级会话。

命令行参数
----------
--privacy            启动时仅显示联系人，不显示消息正文。
--suppress-foreground  微信处于前台时不显示提醒（可选）。
--notify-foreground    兼容旧启动配置；保持微信前台提醒。
--self-test          运行内置逻辑测试并退出。
--integration-test   只读检查能否连接微信会话列表，然后退出。

说明
----
- 程序只能看到微信当前会话列表中已经加载的项目。
- 微信大版本升级后如果修改了辅助功能结构，可能需要同步更新解析规则。
- 本程序不自动发送消息，不修改微信数据。
- 微信列表每 1.5 秒检查一次；遮挡扫描仅在弹窗存在时约每 240 毫秒运行。
- 本机实测普通后台 CPU 约 0.13%，弹窗活动时约 0.42%；不同设备会有差异。
- 工作集内存约 57～66 MB，日志总占用约不超过 4 MB。
- 通知中心功能依赖 Windows 中为本程序启用通知权限。
