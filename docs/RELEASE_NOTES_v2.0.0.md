# WeChat Message Notifier v2.0.0

v2.0.0 在原有本地自定义弹窗基础上增加 Windows 通知中心双通道，并完善点击提醒后的会话直达、前台提醒、服务号解析和多弹窗稳定性。

## 主要更新

- 自定义弹窗负责即时提醒，消息同时静默写入 Windows 通知中心。
- 自定义弹窗创建失败时，由 Windows 系统横幅自动兜底。
- 自定义弹窗 2 分钟后关闭，通知中心历史仍然保留，可由用户手动清除。
- 点击通知中心记录可激活当前提醒器并打开微信。
- 清空通知中心后重置内部消息标签，避免新通知数量沿用旧累计值。
- 点击自定义弹窗后匹配对应一级会话，并通过 UI Automation 与窗口内点击切换聊天内容。
- 修复微信 4.x 报告会话“选择成功”但右侧聊天内容没有刷新的问题。
- 修复服务号聚合行取错摘要、导致顶部新消息可能漏报的问题。
- 微信处于前台时默认继续提醒，避免点击上一条通知后漏掉其他会话的新消息。
- 修复多个置顶弹窗在 60 FPS 动画中反复争夺层级、导致第二条消息闪烁的问题。

## 隐私与安全

- 完全在本机运行，不连接外部服务器，不调用 AI API，不消耗 Token。
- 不读取微信数据库，不读取进程内存，不修改微信数据。
- 联系人和消息正文不会写入程序日志。
- Windows 通知中心历史由当前 Windows 用户的系统通知功能保存。
- 点击自定义弹窗时只向已匹配的可见微信会话行发送一次本地窗口点击，不移动鼠标。

## 性能

- 无弹窗时仍以约 1.5 秒的间隔只读扫描微信会话列表。
- 60 FPS 定时器仅在自定义弹窗存在并需要定位或动画时运行。
- 遮挡检测约每 240 毫秒执行一次，避免跟随动画帧频重复扫描窗口。
- 通知中心同步为单次本地系统调用，不产生持续后台网络或 Token 消耗。
- 日志继续按单文件 1 MB 轮转，最多保留 3 份历史文件。

## 已知限制

- 只能识别微信当前已经加载到会话列表中的项目。
- 微信大版本若改变 UI Automation 结构，解析或跳转规则可能需要同步调整。
- 通知中心记录依赖 Windows 为本程序启用通知权限。
- 相同显示名称的联系人仍可能被视作同一个会话。
