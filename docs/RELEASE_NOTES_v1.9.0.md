# WeChat Message Notifier v1.9.0

发布日期：2026-07-01

## 主要更新

- 消息提醒改为显示 2 分钟后自动关闭；点击提醒仍可打开微信。
- 右上角新增“×”，可只关闭当前提醒而不打开微信。
- 同一联系人连续发消息时更新现有提醒，并重新计算显示时间。
- 使用微信会话 AutomationId、未读数量与内容签名共同判断新消息。
- 增加 5 分钟同联系人同摘要冷却，避免未读计数回跳导致重复提醒。
- 忽略时间文本变化、会话列表短暂缺失等 UI Automation 抖动。
- QQ、Windows 短通知和托盘展开面板遮挡时自动上浮。
- 剪贴板历史等大型面板不会触发上浮。
- 上浮与回落使用约 60 FPS 的非线性阻尼弹簧动画。
- Windows 11 下增加圆角、阴影、轻透明和瞬态亚克力背景。
- 点击提醒打开微信时保留微信原有窗口状态，不会把最大化窗口还原。
- 日志达到 1 MB 后自动轮转，最多保留 3 份历史日志。

## 隐私与安全

- 完全本地运行，不访问网络，不调用 AI API，不消耗 Token。
- 不读取或修改微信数据库。
- 不注入微信进程，不读取微信进程内存。
- 不自动发送微信消息。
- 日志不写入联系人和消息正文，只记录运行状态、字段长度和错误类型。

## 性能说明

- 微信会话列表每 1.5 秒只读检查一次。
- 遮挡检测仅在弹窗存在时运行，频率约为每 240 毫秒一次。
- 60 FPS 定时器只计算动画坐标；没有弹窗时自动停止。
- 日志总占用约不超过 4 MB。

本机 Windows 11、16 逻辑处理器环境的 10 秒采样：

| 状态 | 平均 CPU | 工作集内存 | 私有内存 |
| --- | ---: | ---: | ---: |
| 普通后台运行 | 约 0.13% | 约 66 MB | 约 42 MB |
| 弹窗活动、60 FPS 定时器运行 | 约 0.42% | 约 57 MB | 约 34 MB |

以上数据会随微信会话数量、屏幕数量、置顶窗口和系统负载变化。

## 已知限制

- 微信必须运行并保留可访问的会话列表；完全退出时无法监控。
- 微信关闭到托盘后是否仍可监控，取决于当前微信版本是否保留 UI Automation 会话树。
- 相同联系人和摘要在 5 分钟内只提醒一次。
- 图片、语音和文件只显示微信会话列表提供的摘要。
- Windows 通知中心同步尚未包含在 v1.9.0 中。
